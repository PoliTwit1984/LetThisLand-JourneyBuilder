Journey Design Best Practices

Lifecycle Stages

Start with clear audience segmentation. Every user sits in one of these stages. Build one journey per transition—never overload a single journey with multiple goals.

Pre-Activation

Registered, no session yet

Activated

First session completed

Early Engagement

2–5 sessions, exploring

Progressing

Multi-feature, regular usage

Loyal

6+ features, high retention

At-Risk

Declining sessions, dormancy

Churned

Cancelled or 30+ days dark

Journey Architecture

One journey per lifecycle transition

Pre-Activation → Activated is a journey. Activated → Engaged is a separate journey. Mixing goals kills clarity and makes measurement impossible.

8–20 touchpoints for multi-week journeys

Fewer feels abandoned. More feels spammy. The 45-day playbook targets ~16 touchpoints across 7 phases.

Progressive feature introduction

Follow the natural learning curve: Practice → Session Review → Session Insights → Courses → Range → Target. Don't show Target Mode to someone who hasn't done a session yet.

Decision splits at key behavioral moments

Did they practice? Did they try the feature? Split the path based on the answer. Active users get in-app nudges. Dormant users get push re-engagement.

Triggers & Exit Conditions

Behavioral Triggers (use these)

  • •User completes first session
  • •User tries a new feature mode
  • •No session in 3+ days (dormancy)
  • •User views Session Detail page
  • •Trial claimed / trial expiring
  • •Shot count milestones (100, 500, 1000)

Exit Conditions (always define these)

  • •14+ day dormancy → move to re-engagement journey
  • •Subscription cancelled → exit immediately
  • •User unsubscribes from channel
  • •Journey goal achieved (feature adopted)
  • •User enters a higher-priority journey

Entry Filter Rule

Every journey entry should check: NOT in another active journey to prevent message overlap. A user should never be in two engagement journeys simultaneously.

Cadence Rules

Channel Minimum Spacing Weekly Max Re-entry
Email 3 days apart 3 per week Disabled for engagement
Push 7 days apart 1 per week Disabled for engagement
In-App Event-triggered 1 per session N/A (event-based)
Transactional As needed No cap Enabled

Weekly Anchor Pattern

Email (educate) → In-App (catch in app) → Push (re-engage dormant) → repeat

Email Best Practices

Subject Lines

Length

Max 70 chars • 40–50 ideal for mobile
0 40–50 ideal 70 max

Do

  • • Lowercase, conversational tone
  • • Like a coach texting, not a corporation emailing
  • • Questions: "are you leaving yards on the table?"
  • • Data hooks: "you hit {{total_shots}} shots this week"
  • • Personalization sparingly ({{firstName}} works)

Don't

  • • No exclamation marks
  • • No ALL CAPS
  • • No clickbait
  • • No guilt-trip: "We miss you!"
  • • Don't overuse {{firstName}} in subjects

Copy Structure

2–3 short paragraphs max. Respect the reader's time. Brand voice: warm coach, not salesperson.

1

Acknowledge their situation

Empathy first. Meet the user where they are. Use {{firstName}} once here.

"Hey {{firstName}}, you've been putting in the work with {{total_shots}} shots last week."

2

The insight / value / feature introduction

What the feature does and why it matters to them specifically.

3

The ask (soft CTA)

One clear action. Keep it low-friction.

Bullet Rules

Max 3 bullets. Action-oriented. Under 10 words each. Never guilt-trip: no "We miss you!" or "You're falling behind!"

Design & Layout

Layout Rules

  • • 80:20 text-to-image ratio (spam filter safe)
  • • Mobile-first: single column layout
  • • Table-based HTML for email client compatibility
  • • Dark mode compatible (test prefers-color-scheme)

CTA Button

  • • Rapsodo red #ce2029 on white
  • • Min 44px tap target
  • • 2–4 words, action verb
  • • One primary CTA per email. Optional subtle secondary link.

Additional Details

  • • Preheader: 2–3 words, ALL CAPS (acts as context label in inbox preview)
  • • Footer: social links, unsubscribe via {{unsubscribeUrl}}
  • • {{firstName}} used once (first paragraph). Don't over-personalize.

A/B

Test subject lines always

A/B test subject lines on every send. Even small lifts compound over weeks across thousands of users.

Push Notification Best Practices

When to Use Push

Use Push For

  • Dormancy re-engagement (no session in 3+ days)
  • Data hooks: "Your smash factor was 1.47 last session"
  • Time-sensitive reminders (trial expiring in 2 days)

Never Push

  • Active users already practicing (use InApp)
  • Generic messages ("Check out our app!")
  • During quiet hours (9 PM – 8 AM local time)

Copy Guidelines

Title

Max 50 characters. Punchy, benefit-focused.

your launch angle is trending up

42 chars

Body

Max 100 characters. ONE sentence. Action-oriented.

Tap to see how your last 50 shots compare to your best session.

64 chars

Include specific data when possible. Personalization increases engagement by 30%+. Always give a reason to tap.

Fatigue & Cadence

Fatigue Thresholds

1/wk

Max for engagement journeys

10%

Disable notifications if over-messaged

6%

Uninstall app if fatigued

  • • 2–5 per week is the industry ceiling before fatigue sets in
  • • Quiet hours: no sends 9 PM – 8 AM user local time
  • • Quality over quantity—one good push beats five mediocre ones

In-App Message Best Practices

This is the most powerful channel. 25–40% interaction rate.

Format Selection

Modal ~35% interaction rate
Primary Format

Feature discovery, important announcements, actions requiring a decision.

Use cases: Post-session "check out Session Review" prompts. Feature adoption nudges when user hasn't tried a mode. Decision moments.

Banner ~12.5% interaction rate
Subtle / Non-blocking

Subtle tips, pre-session focus suggestions, non-blocking info.

Use cases: "Today's focus: launch angles" before a session. Quick tips. Less intrusive—doesn't stop the user's flow.

Fullscreen Use rarely
Milestones Only

Major milestones only: 8-week journey completion, first 100 sessions, subscription renewal celebration. Reserve for moments that deserve celebration.

Copy Guidelines

Element Constraint Notes
Title 1–2 lines, under 40 chars Benefit-focused
Body 1–2 sentences, under 100 chars After 3 lines users disengage
CTA Button 1–3 words, UPPERCASE Action verb, one primary CTA only
Dismiss "Not now" or X button Always include. Users who can't dismiss feel trapped.

Personalization Wins

Use session data, shot counts, feature usage. "You hit {{total_shots}} shots—here's your breakdown" >>> "Check out this feature"

Timing & Triggers

ALWAYS triggered by behavioral events, never calendar schedule

Post-session completion = highest intent moment

Play Session v2 end event. Perfect for feature discovery modals.

Pre-session start = good for focus suggestions

Use banner format, not modal. Don't interrupt the intent to practice.

On View Session Detail = natural moment for Session Insights

They're already reviewing data. Low-friction context.

Never

  • • NEVER interrupt mid-session
  • • Wait for session_end or session_start

Pairing Rules

  • • InApp + email same day = fine (different contexts)
  • • InApp + push same day = avoid

Design Specs

Visual

  • • One clear action per message
  • • Rapsodo red #ce2029 for CTA buttons
  • • Min 44px touch area for buttons
  • • High contrast between button and background
  • • Brand-consistent with the Rapsodo app

Technical

  • • Clean HTML, all styles inline
  • • Iterable Side-by-Side editor compatible
  • • Support Handlebars merge tags: {{firstName}}, {{total_shots}}, etc.
  • • Images: PNG/JPEG/GIF, max 5MB
  • • Recommended under 500KB for images

Feature Discovery Pattern

For each new feature introduction, follow this three-channel pattern:

Email Day N

Explain what the feature is and why it matters.

Educate. Set expectation. Give context before they see the feature.

In-App Event-triggered (post-session)

"You just practiced—tap to see your session breakdown"

Catches them at peak intent. Modal format. Highest conversion moment.

Push Day N+3–5 (only if dormant)

"Have you tried Session Review?"

Only fires for users who haven't tried the feature. Re-engagement, not redundancy.

Channel Orchestration

Channel Mix Target

EMAIL 40%
IN-APP 35%
PUSH 25%

Target ratio for engagement journeys. No SMS—not set up for Rapsodo.

The Loop

Email

educate

In-App

catch

Push

re-engage

Email educates the user on the feature or concept. In-App catches them while they're already in the app. Push re-engages dormant users who went silent. Then repeat with the next feature.

Orchestration Rules

Never

Email + push same day. These are both out-of-app interruptions. Stacking them trains users to mute you.

OK

Email + in-app same day. Different contexts (inbox vs. in-product). Users don't perceive this as duplication.

Decision Split Pattern

Behavioral check → Active users → InApp feature prompts
Behavioral check → Dormant users → Push re-engagement

Frequency Caps (Across All Journeys)

Channel Cap Scope
Email Max 3/week Across all active journeys combined
Push Max 2/week Across all active journeys combined
In-App Max 1/session Per app session (not per day)

A/B Testing

  • • Subject lines: Test on every email send. Compounding effect over weeks.
  • • Push copy: Test data-driven messages vs. generic. Measure CTR difference.
  • • InApp format: Test modal vs. banner for the same message. Measure interaction rate + downstream conversion.
  • • Even 5% lifts compound across thousands of users and weeks of the journey.
