1 Overview Stats

Industry benchmarks for sports/fitness push notifications. These numbers set the ceiling and floor for every campaign.

Behavioral Trigger CTR

8–15%

Post-session, milestone, streak

Broadcast CTR

1–2%

Generic sends, feature announcements

iOS Opt-In Rate

60%

Requires explicit permission prompt

Android Opt-In Rate

81%

Opt-in by default (pre-Android 13)

Fatigue Threshold

10%

Disable after 1 irrelevant push/week

Behavioral vs Broadcast

6–8x

CTR multiplier for triggered sends

Key insight: Every push notification either earns the next one or loses a subscriber. Behavioral triggers outperform broadcast by 6–8x because they arrive when the user already has context. One irrelevant push per week causes 10% of users to disable notifications entirely.

3 Character Limits

Stay in the green zone. Users scan notifications in under 2 seconds — truncated text kills comprehension and CTR.

Field Platform Safe Caution Truncated
Title Android ≤ 30 31–40 41+
iOS ≤ 40 41–50 51+
Body Android (collapsed) ≤ 50 51–80 81+
iOS (collapsed) ≤ 90 91–120 121+

Title Example — 33 chars

"Your Session Review is ready"

Android: safe | iOS: safe

Body Example — 46 chars

"47 shots analyzed — see your breakdown"

Android: caution (collapsed) | iOS: safe

Rule of thumb: Write for Android collapsed first (50 char body). If it works there, it works everywhere. Title should always read as a complete thought under 30 characters.

4 Copy Rules

Six copy patterns that work for fitness/sports push. Every notification should fit one of these frames.

Data Hook

Lead with the user's own number. Creates instant relevance.

"47 shots yesterday — ready for more?"

Best for: post-session, weekly summary

Streak

Activate loss aversion. Users protect streaks.

"3 sessions this week. Keep the streak going."

Best for: mid-week re-engagement, habit loop

Feature Discovery

Surface underused features at the right moment.

"Your Session Review is ready"

Best for: Session Review, Target Mode, Stats

Social Proof

Benchmarks against the community. Competitive golfers respond.

"Range Mode users hit 12% more greens"

Best for: feature adoption, mode promotion

Achievement

Celebrate milestones. Positive reinforcement drives return.

"You just passed 1,000 total shots"

Best for: milestone triggers, monthly summaries

Context

Real-world triggers. Weather, time of day, season.

"Perfect range weather today"

Best for: weekend nudge, seasonal re-engagement

5 Anti-Patterns

Every pattern below actively damages opt-in rates and brand perception. Hard no on all of them.

"We miss you!"

Guilt Trip

Manipulative. Users know they haven't opened the app. Saying "we miss you" frames their absence as a failure. Creates negative association with the brand.

"Don't forget to..."

Condescending

Patronizing tone. Adult users don't need reminders framed as parental instructions. Replace with value: "Your session data expires in 48h" instead of "Don't forget to review your session."

"EXCITING NEWS!"

All Caps Spam

Reads as spam. All caps in push notifications triggers the same mental filter as promotional email. If the news is exciting, the content will show it — the formatting doesn't need to scream.

Emoji in fitness apps

-4% CTR

Data shows -4% CTR for emoji in sports/fitness push. Emoji reads as casual/playful — wrong register for performance-oriented users. Rapsodo's audience skews competitive and data-driven.

First name in push

Redundant

A phone is already personal. "Hey Joe, your session..." wastes 10 characters on information that adds zero value. The notification is already on their phone — they know it's for them.

6 Deep Links

Every push must deep link to a specific screen. Opening to the home screen wastes the tap and trains users to ignore future pushes.

Deep Link Screen Use Case
rapsodo://practice Practice Mode Re-engagement, dormancy nudge
rapsodo://session-review Session Review Post-session data hook
rapsodo://session-insights Session Insights Detailed analysis, feature discovery
rapsodo://courses Courses Learning content, onboarding
rapsodo://target-mode Target Mode Feature adoption, social proof
rapsodo://combine Combine Competitive challenge, milestone
rapsodo://stats Stats Dashboard Progress recap, weekly summary
rapsodo://export-video Export Video Swing video sharing, social
rapsodo://subscription Subscription Trial expiry, upgrade prompt
rapsodo://my-bag My Bag Equipment setup, onboarding

Implementation: Set the deep link in Iterable's push template via the Open URL action. The app's URL scheme handler must be registered for rapsodo:// on both platforms.

7 Timing Strategy

When a notification arrives matters as much as what it says. All times in the user's local timezone.

Post-Session

Within 2h

Triggered by session end event. "Your Session Review is ready." Highest CTR window — user still has context from their practice.

Trigger: sessionComplete

Dormancy Re-engagement

6–8 PM local

Evening practice window. User is off work, thinking about range time. Send after 7+ days of inactivity.

Trigger: daysSinceLastSession ≥ 7

Weekend Nudge

Sat 8–10 AM

Saturday morning before the range fills up. Golfers plan their weekend practice early. Context pattern works well here.

Trigger: scheduled + dayOfWeek = Saturday

Frequency Cap

Max 1/week

Hard lifecycle cap. Post-session triggers are exempt (user-initiated context), but scheduled/dormancy sends must not exceed one per week per user.

Iterable: set frequency capping in campaign settings

8 Iterable Implementation Notes

Template fields and configuration for push campaigns in Iterable.

Field Type Notes
title string Notification title. Supports Handlebars: {{shotCount}}
body string Notification body. Handlebars supported. Write for Android collapsed (50 chars).
deepLink URL Set via Open URL action. Use rapsodo:// scheme.
sound string Use default. Custom sounds require app bundle inclusion.
badge integer iOS only. Sets app icon badge number. Use 1 for unread, reset on open.
richMedia URL URL Image/GIF shown in expanded notification. iOS: Notification Service Extension required. Max 10MB.
actionButtons array Up to 3 buttons (iOS) / 3 buttons (Android). Each has title + action (open URL or dismiss).
customPayload JSON Arbitrary key-value pairs passed to the app. Use for analytics tracking, A/B variant IDs, feature flags.

Example Template Payload

{
  "title": "Your Session Review is ready",
  "body": "{{shotCount}} shots analyzed — see your breakdown",
  "sound": "default",
  "badge": 1,
  "deepLink": "rapsodo://session-review",
  "customPayload": {
    "campaign": "post_session_review",
    "variant": "data_hook_v1"
  }
}

Handlebars variables: Iterable supports user profile fields and event data in push templates. Common variables for Rapsodo: {{shotCount}}, {{sessionCount}}, {{streakDays}}, {{lastSessionDate}}, {{totalShots}}. Always set fallback values with {{default shotCount "your"}} syntax.

Frequency capping: Set at the campaign level in Iterable under Sending > Frequency Capping. Configure "max 1 push per 7 days" for lifecycle campaigns. Post-session triggered campaigns should be excluded from frequency capping (set to "no limit") since they are user-initiated context.
