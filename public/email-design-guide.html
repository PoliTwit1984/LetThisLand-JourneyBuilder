2 Layout & Structure

Single-column, table-based layout. Built for every email client from Apple Mail to Outlook 2016.

Structure Rules

  • ▸ Single-column layout, 600px max-width
  • ▸ Table-based construction (not div-based)
  • ▸ Off-white (#F8F9FA) body background, not pure white
  • ▸ Fluid tables with max-width for mobile
  • ▸ Card-based sections for feature highlights

Zero-Scroll Target

The entire message + CTA must be visible without scrolling on mobile. This means:

  • ▸ Headline + 2–3 sentences + CTA button
  • ▸ No hero images above the fold unless they carry data
  • ▸ Total content height under ~500px on mobile

Base Table Structure

<table role="presentation" width="100%" cellpadding="0" cellspacing="0"
       style="background-color: #F8F9FA;">
  <tr>
    <td align="center" style="padding: 24px 16px;">
      <table role="presentation" width="600" cellpadding="0" cellspacing="0"

3 Typography

Web fonts are unreliable in email. Use system font stacks exclusively, and design for fallback rendering.

Font stack: Helvetica, 'Helvetica Neue', Arial, sans-serif — NO web fonts. Google Fonts, custom @font-face, and Typekit all fail silently in most email clients. System fonts render consistently everywhere.

Alignment Rules

  • ▸ Body text: left-aligned always
  • ▸ Headlines: center-aligned only
  • ▸ CTA buttons: center-aligned only
  • ▸ Context labels: left-aligned, above headline

Line Height

Body text uses 1.6x line-height (26px on 16px font). This creates comfortable reading density on mobile without wasting vertical space.

Headlines use 1.2x line-height for tighter visual grouping.

4 CTA Button Design

One primary CTA per email. Personalized CTAs convert 202% better than generic ones. Bulletproof HTML buttons render even when images are blocked.

Button Specs

  • ▸ 6px border-radius (rounded rectangle)
  • ▸ 48px height, 280px min-width on desktop
  • ▸ 100% width on mobile (via media query)
  • ▸ Background: #ce2029 (Rapsodo red)
  • ▸ Text: #ffffff, 16px, bold
  • ▸ 6px radius = 17–55% higher CTR than sharp corners

CTA Rules

  • ▸ ONE primary CTA per email (never two buttons)
  • ▸ Text link for secondary actions only
  • ▸ Personalized CTAs: 202% better conversion
  • ▸ Use action verbs that describe the destination
  • ▸ Never use "Click Here", "Learn More", or "Submit"

Good CTAs

View Your Stats
Try Courses Mode
Review Your Session

Bad CTAs

Learn More
Submit

5 Subject Line Rules

Max 40 characters (visible on mobile). Sentence case, conversational, like a note from a coach. 2–4 word subjects achieve 46% open rates.

Rules

  • ▸ Max 40 characters (ideal: 30–40)
  • ▸ Sentence case (not Title Case)
  • ▸ 2–4 word subjects: 46% open rate
  • ▸ Reference user behavior when possible
  • ▸ No exclamation marks
  • ▸ No ALL CAPS words

Tone

Write like a coach texting after a session — not a marketing team sending a blast. Conversational, specific, useful.

Behavioral references outperform generic feature pitches every time. "Your 10th session deserves a look" beats "Check out these features!" by 2–3x on open rate.

6 Preheader Best Practices

The preheader is the second line of defense after the subject. 50–70 characters that complement (never repeat) the subject line. Lifts open rates 15.5% and CTR 50%.

Rules

  • ▸ 50–70 characters
  • ▸ Complement the subject, never repeat it
  • ▸ Include specific data when possible
  • ▸ Lifts open rates 15.5%, CTR 50%
  • ▸ Hide with zero-width spaces to prevent body text bleed

7 Dark Mode

40%+ of email opens are in dark mode. Design for it from the start, not as an afterthought.

Dark Mode Rules

  • ▸ Use #F8F9FA (off-white), never #FFFFFF
  • ▸ Bold font weights (600+) for body text visibility
  • ▸ Avoid thin hairline borders (disappear in dark)
  • ▸ Use transparent PNGs for logos, not JPGs
  • ▸ Test on Apple Mail dark mode (strictest inversion)
  • ▸ Add color-scheme: light dark meta tag

Client Behavior

  • ▸ Apple Mail: Full color inversion
  • ▸ Outlook (Win): Partial inversion, ignores CSS
  • ▸ Gmail: No inversion (own dark theme)
  • ▸ Yahoo: Full inversion, respects CSS

8 Mobile Optimization

60–70% of email opens are mobile. 80% of users delete emails that render poorly on mobile. Design mobile-first, then verify desktop.

Mobile Requirements

  • ▸ 16px minimum body text (no 12px/14px on mobile)
  • ▸ Full-width CTA buttons on mobile
  • ▸ 44x44px minimum touch targets
  • ▸ 20px side padding on mobile
  • ▸ Stack multi-column layouts to single column
  • ▸ Images: max-width 100%, height auto

The 80% Rule

80% of users who encounter a poorly-rendered mobile email will delete it immediately. There is no second chance.

This means mobile rendering is not a nice-to-have — it is the primary design constraint. If it doesn't look right at 375px wide, it doesn't ship.

Mobile Media Query

@media only screen and (max-width: 600px) {
  /* Full-width layout */
  .email-container { width: 100% !important; }

  /* Side padding */
  .email-content { padding: 24px 20px !important; }

  /* Full-width CTA */
  .cta-button {
    width: 100% !important;
    min-width: 0 !important;
    display: block !important;
    padding: 16px 0 !important;
  }

  /* Responsive images */
  .email-image {
    width: 100% !important;
    height: auto !important;
    max-width: 100% !important;
  }

  /* Stack columns */
  .column { display: block !important; width: 100% !important; }

  /* Prevent font scaling on iOS */
  * { -webkit-text-size-adjust: 100% !important; }
}

9 Anti-Patterns

Common mistakes that destroy open rates, CTR, and deliverability. If you see any of these in a draft, fix them before sending.

Wall of Text

More than 125 words kills engagement. If you need more content, link to a landing page. The email is a door, not the room.

Multiple CTA Buttons

Two or more buttons create decision paralysis. One primary CTA per email. Secondary actions get text links only.

Stock Photography

Generic stock images signal generic content. Use product screenshots, data visualizations, or no images at all. Every pixel must earn its space.

Generic "Learn More" CTAs

"Learn More" tells the user nothing about what happens when they click. Use action verbs that describe the destination: "View Your Stats", "Try Courses Mode".

Identical Visual Structure

If every email looks the same, users stop looking. Vary the layout between plain-text-style, card-based, and data-highlight formats across the journey.

"Click Here" Link Text

Meaningless without context and terrible for accessibility. Screen readers read link text in isolation — "Click Here" says nothing. Use descriptive link text.

Pure White Background

Pure white (#FFFFFF) gets inverted to pure black in dark mode, creating harsh contrast. Use #F8F9FA (off-white) for the email body background.

Time-Based Cadence Only

Sending "Day 3, Day 7, Day 14" emails regardless of user behavior is spray-and-pray. Behavioral triggers (session completed, feature used, milestone hit) outperform time-based cadence by 6–8x.

10 Iterable Implementation

Iterable-specific patterns for building and maintaining lifecycle email templates. Use Side by Side editor, full HTML templates, and Handlebars for personalization.

Editor & Template Rules

  • ▸ Use Side by Side editor (not Drag & Drop)
  • ▸ Templates must be full HTML (not partials)
  • ▸ Use Snippets for reusable components
  • ▸ Header snippet, footer snippet, CTA snippet
  • ▸ Name templates: journey_phase_variant

Common Merge Tags

  • {{firstName}} User first name
  • {{mlm2numSessions}} Total sessions
  • {{total_shots}} Total shots taken
  • {{trialDaysRemaining}} Trial countdown
  • {{lastSessionDate}} Last session date

Conditional Content with Handlebars

<!-- Show different content based on session count -->
{{#if mlm2numSessions}}
  {{#eq mlm2numSessions 0}}
    <p>Start your first session to unlock your stats.</p>
  {{/eq}}
  {{#gt mlm2numSessions 0}}
    {{#lt mlm2numSessions 5}}
      <p>You’ve completed {{mlm2numSessions}} sessions.
         Try adding Range mode next.</p>
    {{/lt}}
    {{#gte mlm2numSessions 5}}
      <p>{{mlm2numSessions}} sessions in — check your
         Insights tab for trend analysis.</p>
    {{/gte}}
  {{/gt}}
{{else}}
  <p>Your practice data is waiting for you.</p>
{{/if}}

<!-- Feature-specific CTA -->
{{#if hasUsedRange}}
  <a href="https://rapsodo.com/insights">View Your Insights</a>
{{else}}
  <a href="https://rapsodo.com/range">Try Range Mode</a>
{{/if}}

Snippet Usage

<!-- Insert reusable header snippet -->
{{snippet "rapsodo_email_header"}}

<!-- Main content goes here -->
<h1>Your session is ready</h1>
<p>{{total_shots}} shots analyzed...</p>

<!-- Insert reusable CTA snippet -->
{{snippet "rapsodo_cta_primary" ctaText="View Your Stats"
         ctaUrl="https://rapsodo.com/session"}}

<!-- Insert reusable footer snippet -->
{{snippet "rapsodo_email_footer"}}

Always set fallback values: Use {{default firstName "there"}} to prevent blank fields when data is missing. Every merge tag should have a graceful fallback. An email that says "Hi , your session..." is worse than no email at all.

11 Performance Benchmarks

Target open rates and CTR by email type. Use these to evaluate campaign performance and identify underperformers.

Email Type Open Rate CTR Notes
Welcome 50–70% 20–40% Highest engagement of any lifecycle email
Onboarding (Day 1–7) 40–60% 10–20% Drops sharply after day 3 without behavioral hooks
Feature adoption 25–40% 5–12% Behavioral triggers outperform time-based 3–5x
Trial midpoint 30–45% 8–15% Data-hook subject lines lift open rate 20%+
Trial expiration 40–60% 10–20% Urgency + loss aversion = highest conversion intent
Re-engagement 15–25% 3–8% Diminishing returns after 2nd re-engagement attempt

Reading this table: Green ranges are strong baselines that well-designed lifecycle emails should consistently hit. Yellow ranges indicate campaigns where copy quality and behavioral targeting create the biggest variance. Red ranges are inherently lower because re-engagement targets disengaged users — but they still deliver positive ROI if the content is relevant to what originally engaged the user.
